- Development

Using word data for the code produced some technicalities and bugs:

  • Typos A negative word will not be recognised if it contains typos (incorrect spelling).
  • Context The program will lack the context of where the negative word appears. For example, the program will be unable to detect irony.

A number of possibilities for developing the software are currently being explored:

  • Text Parsing Other prototypes use external API's for parsing the text highly accurately. Further development could include using models of sentiment analysis for the negative word interactions.
  • File-Based System Interactions of recording degradation from removing words over time, could move into a file-based system.
  • New Words Appear The software would generate new and helpful words within the empty spaces.
